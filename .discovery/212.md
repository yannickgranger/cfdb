## Forbidden moves (from .context/212.md)
- Do not leak snapshot-store semantics into the diff verb. RFC §6 G5 snapshots ≠ keyspaces; `cfdb diff` is keyspace-to-keyspace. The issue-body phrasing "snapshot diff ships with the snapshot store in Phase B" is legacy — interpret as "real diff implementation", not "requires new snapshot store."
- Do not diff the in-memory graph directly. AC3 (determinism) requires diffing sorted canonical dumps (RFC §12.1) or equivalent sorted-JSONL output. In-memory iteration order is not byte-stable.
- Do not place `DiffEnvelope` types in `cfdb-cli`. They are consumed by qbot-core #3736's gate script and possibly by `cfdb classify` (#213). Keep them in `cfdb-query` or `cfdb-core` for downstream deserialization.
- Do not extend `StoreBackend` trait unless strictly necessary. `list_keyspaces`, `canonical_dump`, `schema_version` are already present. Adding new trait methods affects all impls (petgraph, potential HIR backend).
- Do not skip the `Tests:` block. Architects must fill in all four rows (Unit / Self dogfood / Cross dogfood / Target dogfood) in the issue body before an implementation PR can ship. `CLAUDE.md` §2.5 mandates this.
- Do not change exit-0 semantics. AC says "exit 0 regardless of diff size." The consumer gate in qbot-core #3736 decides pass/fail — cfdb diff is informational.

Source: docs/RFC-cfdb.md §12.1 (lines 653-671)
  Package quote: "sorted-JSONL canonical dump is byte-stable across extracts"
  Expansion reason: the forbidden move permits "equivalent sorted-JSONL output" — expansion needed to confirm what "equivalent" means structurally, since it gates whether the diff can iterate `KeyspaceState` directly (with canonical sort keys) vs must parse the `canonical_dump()` string back into Node/Edge values.
  Expanded quote:
    "1. Canonical dump format. `cfdb dump --db <path> --keyspace <name>` emits one JSON object per node and edge, sorted by `(node_label, qname)` then `(edge_label, src_qname, dst_qname)`. Numeric and string fields ordered alphabetically within each object. UTF-8, LF newlines, no trailing whitespace. Each emitted object carries a `kind: "node" | "edge"` discriminator so a stream-parser does not need positional context.
     3. Extractor invariants enforced by architecture test:
       - `BTreeMap` (not `HashMap`) in any collection that feeds the dump
       - Single-threaded write phase (no `rayon` in extractor write path)
       - `slice::sort` (stable), never `sort_unstable`, on any collection that feeds the dump
       - No system clock reads during extraction"
  Updated implication: "equivalent" = any traversal that produces sorted-by-`(label, qname)` / `(label, src_qname, dst_qname)` ordering AND satisfies the BTreeMap/stable-sort/no-clock rules. Both (a) parse canonical_dump() JSONL at cfdb-query level and (b) iterate KeyspaceState with explicit sort at cfdb-petgraph level are permissible. Implementation-level choice left to /prescribe.
