## Forbidden moves (from .context/100.md)

- **Do not fork `LoadError`.** `crates/cfdb-concepts/src/lib.rs:210` is canonical. `load_published_language_crates` must return the SAME `LoadError` type (both `Io` and `Toml` variants). A new `PublishedLanguageLoadError` would create a parallel error vocabulary.
- **Do not re-implement TOML loading from scratch.** `collect_toml_entries` at `crates/cfdb-concepts/src/lib.rs:140` + the `fs::read_to_string` + `toml::from_str` pattern at line 168 is the canonical shape. Reuse the pattern (either verbatim if the file semantics are similar, or mirror the pattern for the single-file `.cfdb/published-language-crates.toml`).
- **Do not land the loader in `cfdb-core` or `cfdb-extractor`.** PR #103 (closing #3) pinned `cfdb-concepts` as the home for concept-file loaders. Anywhere else is a layer violation.
- **Do not bump `SchemaVersion`.** Per issue body "Does NOT bump `SchemaVersion` (additive prop on existing `:Crate` shape)" — the prop is additive on an already-declared node label. No lockstep `graph-specs-rust` fixture bump needed.
- **Do not create a second `.cfdb/*.toml` loader pattern.** The existing `.cfdb/concepts/<name>.toml` pattern loads a directory of per-context files; the new `.cfdb/published-language-crates.toml` is a single file. The loaders differ in that one structural detail — but the TOML parse + error surfacing + `BTreeMap` discipline must match exactly. Do not invent a new one.
- **Do not emit `:Finding`s from `load_published_language_crates`.** Per issue body "Does NOT emit `:Finding`s — that's #48 classifier reading the prop". Loader is pure data → typed API. No classifier logic.
- **Do not add `--cache-dir` / `--published-language-dir` / `--ignore-published-language` CLI flags.** Issue body does not mandate a CLI surface. File a follow-up if a real need emerges.
- **Do not validate consumer-import relationships at load time.** Per issue body "Does NOT validate that declared consumers actually import the crate (static check only — runtime check is #48 scope)". TOML parse is the only validation.
- **Do not ratchet.** cfdb `CLAUDE.md` §3 + global `~/.claude/CLAUDE.md` §6.8 — no baseline / ceiling / allowlist files.
