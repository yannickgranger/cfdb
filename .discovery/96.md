## Forbidden moves (from .context/96.md)

- **Do not re-implement `GitWorktree`.** The RAII guard at `crates/cfdb-cli/src/commands.rs:142-195` is the single tempdir-backed worktree manager; URL clone must layer above it, not alongside.
- **Do not fork `short_rev` keyspace-naming.** Current impl at `commands.rs:119-127` is the canonical short-rev → keyspace mapper. Reuse it for URL@SHA keyspace naming.
- **Do not add an unconditional `git2` dep to `cfdb-cli`.** `git2` is already a dep but behind the `git-enrich` feature (PR #105 / commit `242ef1e`). If URL clone needs a library path (vs. shelling `git clone`), preserve the feature-gating pattern — do not make `git2` unconditional.
- **Do not drop the subprocess-contract for same-repo `--rev`.** PR #123 body explicitly states "No git library dep added — subprocess is the contract" for the same-repo path. Option W can use subprocess `git clone` (inherits ambient credentials) or feature-gated `git2` for the URL path, but must not break the same-repo subprocess invariant.
- **Do not write cache files inside `target/` or `tempdir`.** AC-3 requires persistent cache across runs. Likely location: `~/.cache/cfdb/extract/<sha256(url)>/<sha>/` or configurable via `--cache-dir`. Subject to prescription.
- **Do not hand-forge `.cfdb/current.sha` or commit it.** Per council-cfdb-wiring, `.cfdb/current.sha` is per-workspace state, `.gitignore`'d already at line 4.
