- Do not embed `fix_skill` / routing hints in finding rows. RFC §A2.3 + line 919 council BLOCK-1 / solid-architect. Routing is external via `.cfdb/skill-routing.toml`, read by the CONSUMER skill (qbot-core #3736 gate, `/operate-module`, etc.). Architecture test `finding_no_skill_field` at `crates/cfdb-query/tests/finding_no_skill_field.rs` enforces this — do not bypass.
- Do not re-implement classifier logic. The canonical entry points are `populate_findings_by_class` (`crates/cfdb-cli/src/scope.rs:154`) and `class_empty_bucket_note` (same file, ~line 291). A second population path in a new `classify.rs` module would be split-brain — the new verb must CALL the shared function, restricting its output to diff-qnames afterwards (or by passing a filter parameter if one is added).
- Do not add rules to synthesise `random_scattering` or `unwired` buckets. Per session `2026-04-15-3729-cfdb-scope-shipped`: those buckets are legitimately empty on v0.1 (enrichment producers not shipped yet). Empty-bucket warnings via `class_empty_bucket_note` are the documented behaviour. The new verb must surface the same warnings.
- Do not route ContextHomonym to `/sweep-epic` in any code or doc. Council guardrail — mechanical sweep on a cross-context homonym deletes legitimate bounded-context isolation. `.cfdb/skill-routing.toml` already encodes the correct routing (`ContextHomonym → /operate-module council-required`). Do not override.
- Do not place envelope types (e.g. `ClassifyEnvelope`, `ClassifyRequest`, etc.) in `cfdb-cli`. They are wire contracts deserialised by qbot-core #3736; must live in `cfdb-query` alongside `ScopeInventory` + `Finding` + `DebtClass`. If a new type is needed at all, follow the `DiffEnvelope` placement pattern established by #234 (`crates/cfdb-query/src/diff.rs`).
- Do not cite `RFC-029 v0.2 §A2` or `docs/RFC-cfdb-v0.2-addendum-draft.md`. The addendum was merged into `docs/RFC-cfdb.md` §A2 via PR #198 / commit `6cc5296`; the draft file no longer exists on develop. Cite the canonical merged locations (§A2, §A2.2, §A2.3, line 919).
- Do not skip the `Tests:` block. `CLAUDE.md §2.5` requires four rows (Unit / Self dogfood / Cross dogfood / Target dogfood). Issue body already carries it — honor the prescription, don't re-draft.
- Do not change exit-0 semantics. The verb is informational; the consumer gate (qbot-core #3736) decides BLOCK/WARN. Exit 0 regardless of class counts is the contract.
- Do not classify on diff-JSON alone without a keyspace. RFC §A2.2 binds: the classifier is a query over an enriched graph. The verb's shape is `--db <dir> --keyspace <ks> --restrict-to-diff <diff.json>` — all three flags load-bearing. A `--input <diff.json>`-only variant is forbidden.
