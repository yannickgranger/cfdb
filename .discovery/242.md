━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Two ambiguities surfaced that prescription must resolve before the fix is written:

**Ambiguity A — root cause location.** The reproduction shows three hypotheses (Chain 2 ⚠️ block above):
  (A) `state.graph.edges(idx)` returns empty for every `idx` enumerated by `candidate_nodes` — i.e. `graph.add_edge` never ran, or ran against different NodeIndex values.
  (B) Every `e.weight().label` comparison fails in `edge_label_matches` (ruled out for anonymous `-[r]->` queries that also return 0).
  (C) NodeIndex desync between `ingest_nodes` (populates `id_to_idx`) and `ingest_edges` (resolves src/dst via `id_to_idx`). StableDiGraph is index-stable under removals but `ingest_one_edge` dereferences `id_to_idx.get(&edge.src)` which MUST succeed at edge-ingest time for the edge to be added to the graph. If the id isn't in `id_to_idx`, the edge is silently dropped (EmptyResult warning) per the forbidden-move lesson.

  Prescription decision needed: which hypothesis does the fix investigate first, and what is the minimal repro test that disambiguates?
  Disambiguation probe (observable): `state.graph.edge_count()` and `state.edge_labels` are `pub(crate)` on `KeyspaceState`. A test that loads cfdb's own keyspace and reads these fields would reveal whether edges are in the graph at all (`edge_count > 0`) or whether the drop/desync fires at ingest (`edge_count == 0`). This is a fact about the shape of the keyspace, not a prescription — /prescribe decides whether that probe is the fix's first step.

**Ambiguity B — silent-empty warning expansion.** RFC-cfdb.md §6 mandates `{rows, warnings}` precisely to eliminate silent-empty failure modes. The `UnknownEdgeLabel` warning covers one class; the current `EmptyResult` warning (fired only when `rows.is_empty() AND no Unknown* warning`) fires for the user-facing output but does not itself disambiguate "zero rows matched" vs "evaluator bug". Prescription decision needed: whether the fix extends warning coverage to "edge label recognized but zero rows" (observability improvement in-scope) or defers that extension to a separate follow-up issue (observability out-of-scope for this bug).
  (Forbidden moves bullet: "Do not skip the UnknownEdgeLabel warning audit" — the audit is due; the decision about whether to EXTEND the warning coverage is up to /prescribe.)

These ambiguities are intentionally left for /prescribe — discovery has the facts; prescription picks the shape.
