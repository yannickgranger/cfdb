### `ParamResolveError` (enum with variants `UnknownForm`, `UnknownContext`, `Io`, `Toml`)

- **CREATE** new `pub(crate) enum ParamResolveError` in `crates/cfdb-cli/src/param_resolver.rs`
- **BOUNDED_CONTEXT:** `cfdb-cli` — this type belongs here because it models errors of the CLI-arg `--param` parsing workflow, which is a CLI-layer concept. Neither `cfdb-core` (domain) nor `cfdb-concepts` (TOML loader) owns "CLI arg form parse failed". Council RATIFIED §9 names `cfdb-cli` as canonical authority. RFC-034 §3.1 places the type here.
- **Why create:** independent decision — CLI-arg parsing errors will evolve separately from TOML-loader errors (`cfdb_concepts::LoadError`) and from the tool-local triggers errors (`check_prelude_triggers::toml_io::LoadError`).
- **Not duplication because:** if `cfdb_concepts::LoadError` changes (e.g., adds a new variant for malformed `canonical_crate`), `ParamResolveError` should NOT change — it wraps `LoadError` via `#[from]` on `Io`/`Toml` (one-shot conversion at the boundary), not by reflecting every upstream variant. The test "if the original changes, must this new code change too?" → NO for most upstream changes; YES only for the narrow `{ path, source: io::Error }` / `{ path, source: Box<toml::de::Error> }` shape that the RFC §3.1 signature locks in.
- **Name collision check (grep):** `rg "pub\s+(struct|enum|type)\s+ParamResolveError" --type=rs crates/ tools/` → zero hits. No collision.
- **Name collision check (cfdb, pinned keyspace 0845a32b3d72):** `cfdb list-items-matching --db .cfdb/db --keyspace cfdb --name-pattern "^ParamResolveError$" --kinds Struct,Enum,Trait` → zero rows. No collision.
- **Searches exhausted:** (a) `rg "ParamResolveError" --type=rs` — only RFC + council hits, zero Rust source; (b) `rg "fn resolve_param|fn resolve_params" --type=rs` — zero; (c) `rg "\.parse::<.*Param" --type=rs` — zero param-shaped FromStr impls. No existing abstraction covers this responsibility.
- **Shape (per RFC §3.1):** `#[derive(Debug, thiserror::Error)]` with four variants:
  - `UnknownForm { form: String }` — prefix on `<name>:<form>:<value>` was not one of `context`/`regex`/`literal`/`list`
  - `UnknownContext { name: String }` — `context:<name>` referenced but `.cfdb/concepts/<name>.toml` not in `ConceptOverrides::declared_contexts()`
  - `Io { path: PathBuf, #[source] source: std::io::Error }` — propagated from `LoadError::Io` at the boundary
  - `Toml { path: PathBuf, #[source] source: Box<toml::de::Error> }` — propagated from `LoadError::Toml` at the boundary
- **Pattern precedent (NOT a copy):** `cfdb_concepts::LoadError` at `lib.rs:216` uses the same `{ path, source }` `thiserror`-derived shape. Mirroring the shape without copying the type is the documented DDD precedent (see Discovery DECISION ARCHAEOLOGY — `tools/check-prelude-triggers::LoadError` mirrored `cfdb_concepts::LoadError` under Forbidden-move-#8 justification; that justification does NOT extend to `cfdb-cli` which WILL add `cfdb-concepts` as a direct dep in this slice — hence DO NOT reuse `LoadError` name, use `ParamResolveError`).
