## Release mode required

The test is run in `--release` because `ra-ap-hir` contains `never!`
debug-only assertions that fire on certain real-world code shapes
(e.g. ra_ap_hir_ty/src/infer/cast.rs:533 — unsized struct handling).
In debug these are `panic!`; in release they degrade to
`tracing::error!` — the correct behaviour for a production
extractor. RFC-029 §A1.2 line 109 flagged ra-ap-* as a moving
target with ~4 breaking changes / year; this is one such known
instance.

The test is marked `#[ignore]` so `cargo test --workspace` stays
fast. Slice 4 (#86) will wire a CI dogfood job that runs it in
release mode against cfdb itself.
