## Protocol dry-run

Per docs/ra-ap-upgrade-protocol.md §2, the six-step procedure is:

  1. Bump all 12 ra-ap-* pins simultaneously to the new 0.0.N
     bundle number (ra-ap-hir is authoritative; others match).
  2. Bump ra-ap-rustc_type_ir to its current HEAD (independent
     2-3×/week cadence).
  3. Run `cargo test --workspace --all-features`
     plus `ci/determinism-check.sh` — byte-identical canonical
     dumps across two extracts (G1).
  4. Run Pattern B/C reproduction tests (currently mapped to the
     existing arch-ban rule suite as a proxy).
  5. If GREEN: merge the chore branch. The HIR CI gate is allowed
     to consume the new version.
  6. If RED: file a `chore(ra-ap): compat issue with 0.0.N` ticket;
     pin stays at last-known-good; triage within 7 days.
