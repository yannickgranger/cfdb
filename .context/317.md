# Context package — Issue #317

## Issue summary
`cfdb-extractor` emits IMPLEMENTS / IMPLEMENTS_FOR / RETURNS / TYPE_OF edges whose dst qname is not in `Emitter::emitted_item_qnames` (foreign traits like `std::fmt::Display`, `serde::Serialize`). The petgraph ingest layer drops every such edge: 0 IMPLEMENTS edges survive on cfdb-self despite dozens of `impl Display`/`impl FromStr` blocks. Fix: synthesize minimal `:Item { name, qname, kind, crate }` nodes for any qname referenced as edge dst, run as a post-walk pass in `extract_workspace`, dedupe via `BTreeMap<String, &'static str>` with stronger-evidence-wins promotion (IMPLEMENTS-as-trait > IMPLEMENTS_FOR-fallback-struct).

## Forbidden moves
- **Do NOT add `:Item.source` / `:Item.external` / any new attribute on `:Item`.** RFC-039 withdrawal `fdaf333` rationale: "no new schema attribute, no new enum, no new vocabulary."
- **Do NOT bump `cfdb_core::SchemaVersion`.** Adding optional-attribute-bearing nodes to an existing label is non-breaking. Bumping triggers a graph-specs-rust lockstep PR (CLAUDE.md §3) that is unwarranted here.
- **Do NOT author an RFC.** Bug fix routes through `/work-issue-lib`, not RFC pipeline (CLAUDE.md §1). RFC-039 was already withdrawn for this reason.
- **Do NOT reintroduce the RFC-039 framing ("foreign things to synthesize stubs for").** Foreign traits are our *dependency surface*, not stubs. The minimal node is a real `:Item` whose body fields (file, span, visibility, methods) happen to be absent because we didn't walk its source. Absence of body fields IS the discriminator — no flag needed.
- **Do NOT mark the synthesized nodes with a special kind value (e.g., `kind="external"`).** Kind comes from the edge-label grammar (IMPLEMENTS dst → trait; IMPLEMENTS_FOR/RETURNS/TYPE_OF dst → struct fallback).
- **Do NOT make the existing scope split-brain worse.** No new resolution points in `cfdb-extractor` for qname formula or kind mapping; reuse `cfdb-core::qname::*` helpers (RFC-037 §3.8).
- **Do NOT touch test-helper `build` duplication in cfdb-petgraph/src/enrich/{concepts,rfc_docs}/tests.rs** — out of scope (separate audit finding).
