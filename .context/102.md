# Context package — Issue #102

## Issue summary

Feat: Trigger T3 — new `cfdb check --trigger T3` CLI surface that emits one row per distinct `:Item.name` appearing in items in ≥2 distinct `:Crate`s, with an `is_cross_context: bool` column comparing the bounded_contexts of those crates. T3 is explicitly the RAW detection; #48 classifier consumes T3 rows to label them as `DuplicatedFeature` (same context) or `ContextHomonym` (cross-context).

## Doctrinal layer

- **`docs/RFC-cfdb-v0.2-addendum-draft.md` §A2.1 lines 182–184** — canonical 6-class `DebtClass` taxonomy. Class 1 (DuplicatedFeature) and class 2 (ContextHomonym) are the two classifications T3's raw rows feed into. Implication: T3's `is_cross_context: bool` column IS the same-context-vs-cross-context discriminator that the classifier uses — prescription must ensure the semantic matches what #48's `classifier-duplicated-feature.cypher` and `classifier-context-homonym.cypher` already compute, so downstream consumers can use either path consistently.
- **`docs/RFC-cfdb-v0.2-addendum-draft.md` §A2.2** — classifier pipeline. `:Item.bounded_context` (populated by `enrich_bounded_context`) is the authoritative per-item context attribution. T3's `contexts[]` column reads these — same data the classifier reads.
- **`examples/queries/hsb-by-name.cypher`** — existing Pattern A raw split-brain detector, RESTRICTED to `struct/enum/trait` kinds (no fn/method — "method-name collisions are semantic noise"). The `crates[]` column is already populated. T3 may extend the shape (add `contexts[]` + `is_cross_context`) or emit a sibling query file. Header note in hsb-by-name.cypher: *"DO NOT route cross-context candidates to /sweep-epic based on this query alone during the v0.1 → v0.2 window."* Implication: T3 is the v0.2 form of hsb-by-name with bounded-context awareness baked in — not a parallel rule, but the evolution that closes the v0.1→v0.2 safety gap mentioned in the hsb-by-name.cypher header.
- **`CLAUDE.md` §3** — dogfood enforcement. Determinism CI + cross-dogfood + no ratchets apply.
- **`CLAUDE.md` §2.5** — new-capability 4-row `Tests:` block.
