## Doctrinal layer

- **docs/RFC-cfdb.md §6 (API contract)** — "`query_raw(keyspace, cypher, params, sets?) -> {rows, warnings}` … warnings catch label/attr/edge typos and 'schema mismatch' vs 'empty result' — silent-empty was the #1 LLM-consumer failure mode." **Implication:** the failing queries are within spec; returning 0 for a schema-valid edge pattern is exactly the "silent empty" failure mode the RFC was architected to eliminate. A warning emitted for known-label edges returning 0 rows would have caught this earlier.
- **docs/RFC-cfdb.md §12.1 (Determinism G1)** — "Same workspace SHA + same schema version → byte-identical graph." **Implication:** G1 governs extract determinism, not query correctness. Passing G1 does not prove the evaluator sees edges that the canonical JSONL dump contains. Bug is below G1's ceiling.
- **specs/concepts/cfdb-petgraph.md** — any new pub type in this crate requires a spec entry (user memory `feedback_graph_specs_pub_types.md`). Fix is read-side only so likely no new pub types, but if one is introduced, the spec must land in the same PR.
- **CLAUDE.md §2 rule 8 (no metric ratchets)** — applies if the fix is tempted to add a baseline/allowlist for queries known to return 0; forbidden. Fix the evaluator, not the threshold.
