# Spec: cfdb-extractor-php

PHP `LanguageProducer` MVP (RFC-041 Phase 2 / issue #264). Walks a Composer-style PHP project via tree-sitter-php and emits the v0.1 cfdb fact set onto the existing closed-set schema vocabulary in `cfdb-core::schema::labels` — no new `:Item.kind` values are introduced (Published Language invariant per RFC-041 §4). Operationally a cfdb-monolith member crate per `.cfdb/concepts/cfdb.toml`; gated behind `lang-php` in `cfdb-cli` so default builds drop the entire tree-sitter dep tree.

## PhpProducer

Reference implementation of `cfdb_lang::LanguageProducer` for PHP workspaces. `name()` returns `"php"` (matching the `lang-php` Cargo feature on `cfdb-cli`). `detect()` is the marker-file check `workspace_root.join("composer.json").is_file()` — Composer's PSR-4 manifest is the canonical PHP project marker. `produce()` walks `**/*.php` under workspace_root (skipping `vendor/`), parses each file via tree-sitter-php's `LANGUAGE_PHP` constant (0.23 ABI shared with `cfdb-extractor-ts`), and emits sorted-canonical `(Vec<Node>, Vec<Edge>)`. Any tree-sitter parse failure or I/O error wraps to `LanguageError::Parse { producer: "php", ... }` / `LanguageError::Io`. Schema mapping decision (preserves the closed-set invariant; documented in the crate-root doc + `Cargo.toml` header): PHP `namespace` → `:Module` node; PHP `class` / `interface` / `trait` → `:Item { kind: "trait", php_construct: <ast-kind> }` (the `php_construct` prop is the disambiguation seam for cypher queries that need to distinguish the three); PHP `method` / `function` → `:Item { kind: "fn" }`; PHP `const` → `:Item { kind: "const" }` (reserved; MVP doesn't yet emit class constants).
