# Spec: cfdb-recall

The recall gate for the cfdb extractor. Compares the set of public items `cfdb-extractor` emits for a crate against the set `rustdoc --output-format=json` reports, and verifies the ratio meets the threshold declared in `DEFAULT_THRESHOLD`. Gated behind the `runner` feature when the binary-or-rustdoc-json machinery is needed (PR #33 landed this split).

## AuditList

The set of items the recall gate should ignore when computing the denominator — typically items generated by macros the extractor cannot see (`define_id!`, `#[derive]` expansions). Audit list members are subtracted from BOTH numerator and denominator — they are outside the measurement surface, not "present but unmatched".

## PublicItem

A normalised reference to a public item — its fully-qualified name from the crate root. Equality is keyed on qname alone; item kind is intentionally dropped from the compare because qnames are unique across kinds within a crate.

## RecallReport

The result of a single-crate recall computation — total public items, adjusted denominator (after audit carve-out), matched count, missing items, and the audit-list entries actually present in the public set.

## GroundTruthError

Error type for the `rustdoc-json` + `rustdoc-types` adapter path. Gated behind the `runner` feature; library consumers that only call `project_rustdoc_paths` do not link against it.
