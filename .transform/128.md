## Integration protocol (main agent)

2. Main agent fetches each subunit branch into the integration worktree (`.claude/worktrees/128-godfile-split`, branch `work/128-godfile-split`).
3. Main agent merges / cherry-picks commits from each subunit branch. All 4 branches touch disjoint files (different crates), so no merge conflicts expected.
4. Main agent runs **one** `cargo check --workspace --all-targets --all-features` to validate.
5. If check fails, main agent reads diagnostics, applies localized fixes, re-runs. Does NOT spin up sub-agents for compile fixes.
6. Main agent runs `cargo test --workspace --all-features` once. Asserts test counts match baseline.
7. Main agent re-extracts fact graph: `cfdb extract --workspace . --db .cfdb/db-post --keyspace cfdb && cfdb dump --db .cfdb/db-post --keyspace cfdb | sha256sum`. MUST equal `3e73ba98...`. If not equal, diff and fix.
8. Main agent runs `quality-architecture --workspace .` — asserts 4 god-files remain (the 4 drift files), none from the 8 original.
9. Main agent runs both graph-specs passes (from #137's CI invocation). Both must exit 0.
10. Main agent commits the integration, pushes, opens PR via `/ship`.
